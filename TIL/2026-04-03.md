# TIL: 2026-04-03

**Topic:** OpenClaw exec loopback: legacy-role fallback prevents pairing-required errors after 2026.3.31

**What:** After the 2026.3.31 OpenClaw update, local exec and node clients started failing with `pairing required` errors (close 1008) during loopback scope-upgrade pairing. The fix restores a legacy-role fallback for empty paired-device token maps and allows silent local role upgrades, so the exec path no longer breaks when upgrading from legacy paired-device configurations.

**Why it matters:** If you self-host OpenClaw and noticed exec or node client operations suddenly requiring pairing after a recent update, the root cause was an overly strict token map check. The fix is automatic on upgrade — no manual migration needed — but understanding the fallback chain helps debugging similar future failures.

**Rule of thumb:** When exec calls start failing with pairing errors on a previously working setup, check whether a recent OpenClaw update changed the role fallback or token map validation logic. Run `openclaw doctor --fix` to clean up legacy config artifacts.
